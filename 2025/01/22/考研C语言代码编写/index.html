常见的printf格式化占位符

字符和字符串

  • %c：输出一个字符。
  • %s：输出一个字符串。

整数

  • %d 或 %i：输出有符号十进制整数。
  • %u：输出无符号十进制整数。
  • %o：输出无符号八进制整数。
  • %x 或 %X：输出无符号十六进制整数（%x 输出小写字母，%X 输出大写字母）。
  • %p：输出指针（地址）。

浮点数

  • %f：输出十进制定点表示法的小数。
  • %e 或 %E：输出指数表示法的小数（%e 使用小写 ‘e’，%E 使用大写 ‘E’）。
  • %g 或 %G：输出较短的小数或指数表示法（%g 自动选择 %f 或 %e，%G 自动选择 %f 或 %E）。

宽度和精度

  • *：指定宽度或精度为变量。
  • width：指定最小字段宽度。
  • .precision：对于 %f、%e、%E、%g、%G，指定小数点后的位数；对于 %s，指定最大字段宽度；对于整数，指定最小数字位数。

长度修饰符

  • h：与 %d、%i、%o、%u、%x 或 %X 一起使用，表示 short int 或 unsigned short int。
  • l（小写的 L）：与 %d、%i、%o、%u、%x 或 %X 一起使用，表示 long int 或 unsigned long int；与 %e、%f、%g 一起使用，表示 double。
  • ll 或 LL（两个小写的 L）：与 %d、%i、%o、%u、%x 或 %X 一起使用，表示 long long int 或 unsigned long long int。
  • L（大写的 L）：与 %e、%f、%g 一起使用，表示 long double。

递归函数应用

gets和fgets不同

fgets 和 gets 都是 C 语言中用于从标准输入（通常是键盘）读取字符串的函数，但它们之间存在一些关键的不同点：

安全性：
gets 函数非常不安全，因为它不会检查目标缓冲区的大小，这可能导致缓冲区溢出，使得程序崩溃或者被恶意代码利用。由于这个原因，gets 在 C11 标准中被弃用，并且不推荐使用。
fgets 函数则相对安全，因为它允许指定缓冲区的大小，并且确保不会超出这个大小。它会在读取到缓冲区大小减一的字符数或者遇到换行符时停止读取，并在字符串末尾自动添加一个空字符（\0）作为字符串的结束标志。
参数：
gets 只接受一个参数，即用于存储输入字符串的字符数组。
fgets 接受三个参数：第一个是字符数组，用于存储输入的字符串；第二个是缓冲区大小（包括最后的空字符）；第三个是输入流，通常是 stdin。
换行符处理：
gets 会丢弃输入字符串末尾的换行符。
fgets 会将换行符也作为字符串的一部分读入缓冲区。如果需要，用户可以手动移除它。

    局部变量的值.

结构体知识点


结构体对齐三大法则：

  • 1.自然对其规则：自然对其要求每个成员的地址是其数据类型大小的整数倍，
    例如int会被对齐到4字节边界；double会被对齐到8字节边界。
  • 2.填充字节：如果一个成员的大小不是对齐边界的整数倍，那么编译器
    通常会在后面添加一些字节来，以保证下一个成员能够正确对齐
  • 3.结构体的总大小：结构体的总大小通常等于其最大成员大小的整数倍，这是
    为了确保结构体数组中的每个元素按照相同的规则进行排序。
  • 这是s1的内存模型，首先大小为18字节；根据结构体总大小为最大成员的整数倍。满足条件
    根据首地址计算出每个成员的首地址，均满足自然对其规则。地址能够整出成员变量占用的内存。
  • 这里主要是看s2的填充字节的作用，如果直接在sex的地址后面立即存入age的地址，那么不满足自然对其规则
    所以需要在sex的地址后面进行填充，从而使得age满足自然对齐规则。
