[野火]FPGA Verilog开发实战指南——基于Altera EP4CE10 征途Mini开发板

野火征途Mini FPGA开发板，采用Altera EP4CE10F17C8作为主控，包含了FPGA入门必备的硬件，是FPGA入门的性价比之选。

  • 1. 初识Verilog HDL

reg 变量

  • 作用域：在module的括号外面声明的reg变量是全局变量，它们可以在整个编译单元中被访问和修改，这意味着它们可以在多个模块之间共享。
  • 存储类型：reg变量通常用于表示在过程块（如always块）中被赋值的信号，它们可以存储值。
  • 持久性：reg变量的值在仿真过程中会持续存在，除非被显式地改变。
  • 示例：
    reg global_reg;

module my_module();
// …
endmodule

parameter 变量

作用域：parameter变量是在模块内部声明的，它们的作用域仅限于声明它们的模块内部。
常量：parameter用于定义常量值，这些值在仿真期间是固定不变的，不能被修改。
用途：parameter常用于定义模块的配置参数，如计数器的位宽、延迟时间等。
示例：
module my_module();
parameter PARAMETER_WIDTH = 8;
// …
endmodule

主要区别

  • 作用域：reg是全局的，而parameter是局部的。
  • 值的变化：reg变量的值可以改变，而parameter一旦定义后其值是固定的。
  • 用途：reg用于存储信号值，parameter用于定义模块的配置参数。
  • 位置：reg通常在模块外部声明（除非是在模块内部用于局部变量），而parameter总是在模块内部声明。
  • 总结来说，reg变量用于在仿真过程中存储和表示可变的数据，而parameter变量用于定义仿真期间不变的常量值，通常用于模块配置。正确使用它们可以增强代码的可读性和可维护性。

仿真的时候不要在模块内部用变量仿真

在仿真（testbench）环境中，你确实可以在模块外部使用reg类型的变量来模拟时钟和其他控制信号。实际上，这是在编写Verilog测试模块时的常见做法。

在仿真环境中，reg类型用于表示可以由过程块（如initial或always块）驱动的变量。以下是一个典型的测试模块示例，它展示了如何在模块外部使用reg类型的变量来生成时钟信号：

`timescale 1ns/1ns

// 测试模块
module testbench;

// 声明用于仿真的reg类型变量
reg clk;
reg reset;
wire output_signal;

// 实例化待测试的设计单元（DUT）
design_under_test u_dut (
    .clk(clk),           // 连接到DUT的clk端口
    .reset(reset),       // 连接到DUT的reset端口
    .output(output_signal)  // 连接到DUT的output端口
);

// 初始化和时钟生成
initial begin
    clk = 0;
    reset = 1;  // 初始复位信号
    #20 reset = 0;  // 20纳秒后释放复位
    // ... 其他初始化代码 ...
end

// 时钟生成逻辑
always #5 clk = ~clk;  // 每5纳秒翻转时钟，生成10纳秒的时钟周期

// ... 测试逻辑 ...

endmodule
在这个例子中，clk和reset都是reg类型的变量，它们在测试模块的initial和always块中被驱动。clk用于模拟时钟信号，而reset用于模拟复位信号。这些信号随后被连接到待测试的设计单元（design_under_test）的相应端口。

这种方法在仿真环境中是完全可以接受的，因为在仿真环境中，你通常需要模拟外部信号的行为，而这些信号在硬件实现中是由实际的时钟源或控制信号驱动的。在仿真中，使用reg类型的变量来模拟这些信号是标准做法。

always用法

always中如果想给变量赋值的时候，变量必须得是reg类型
在Verilog中，always块是一个基本的结构，用于描述硬件的行为。always块可以与以下几种类型的敏感信号列表一起使用，以定义不同的硬件行为：

  1. 时序逻辑（使用@）
    always @(posedge clk or negedge rst_n) begin
    // 时序逻辑代码
    if (!rst_n) begin
    // 异步复位逻辑
    end else begin
    // 时序逻辑，通常在时钟的上升沿触发
    end
    end
    这里的posedge clk表示时钟的上升沿，negedge rst_n表示复位信号的下降沿。

  2. 组合逻辑（使用@()）
    always @(    ) begin
    // 组合逻辑代码
    out = a & b; // 例如，out是a和b的逻辑与
    end
    这里的@(*)是一个通配符，表示这个块对所有的输入信号变化都敏感。

  3. 电平敏感（不推荐使用）
    always @(signal) begin
    // 电平敏感逻辑代码
    end
    这种方式不推荐，因为它可能导致不可预测的行为，而且不符合实际硬件的工作方式。

注意事项：
always块内不应包含时间延迟（如#10），除非是在测试代码（testbench）中模拟时序。
在描述时序逻辑时，确保使用非阻塞赋值（<=）以避免引入不必要的竞争条件。
在描述组合逻辑时，使用阻塞赋值（=）。
示例：
以下是一个简单的时序逻辑示例，其中包含一个同步复位：

reg [3:0] count;
always @(posedge clk or negedge rst_n) begin
if (!rst_n) begin
count <= 4’b0; // 异步复位到0
end else begin
count <= count + 1; // 每个时钟上升沿递增
end
end
这个示例展示了一个简单的组合逻辑：

wire out;
reg a, b;
always @(*) begin
out = a & b; // out是a和b的逻辑与
end
使用always块时，务必清楚你正在描述的是时序逻辑还是组合逻辑，并相应地编写代码。
