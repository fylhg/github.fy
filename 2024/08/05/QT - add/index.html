QMainWindow和QAction

QMainWindow是主窗口，在对主窗口进行设计的时候，需要主菜单上的工具栏，菜单栏，按钮的下来菜单，组件的快捷菜单都需要用到QAction类,我们可以设计多个QAction创建菜单和工具按钮。

工具栏QToolBar类

通过QAction创建工具栏中的按钮，工具栏对应的就是QToolBar类。
在ui界面项快速创建工具栏可以右击鼠标然后点击”Add Tool Bar”即可。
QToolBar的主要属性：

属性名称 属性值类型 含义和作用
movable bool 工具栏是否可以动
allowedAreas Qt::ToolBarAreas 工具栏可以放置的窗口区域
orientation Qt::Orientation 工具栏的方向：水平、垂直
iconSize QSize 图标大小
toolButtonStyle Qt::ToolButtonStyle 工具按钮样式
floatavle bool 工具栏是否可浮动

ToolButtonStyle:

  • Qt::ToolButtonIconOnly:只显示图标
  • Qt::ToolButtonTextOnly:只显示文字
  • Qt::ToolButtonTextBesideIcon:文字显示在图标旁边
  • Qt::ToolButtonTextUnderIcon:文字显示在图标下面
  • Qt::ToolButtonFollowStyle:由QStyle样式定义

创建右击快捷菜单

每个继承QWidget的类都有customContextMenuRequested()信号
要使QWidget的组件可以发射这个信号需要设置其contextMenuPolicy属性：

  • void QWidget::setContextMenuPolicy
    有以下几种枚举值：
    • Qt::NoContextMenu:组件没有快捷菜单，由父容器处理
    • Qt::PreventContextMenu:阻止快捷菜单，并且右击鼠标事件不会交给父容器处理
    • Qt::DefaultContextMenu:默认快捷菜单
    • Qt::ActionsContextMenu:自动根据QWidget::actions()返回的Action创建并显示快捷菜单
    • Qt::CustomContextMenu:组件发射 customContextMenuRequested()信号，自己写槽函数创建菜单

QTreeWidget

QTreeWidget是一种Item widget组件，类似的可以参考QListWidget它也是Item widget组件。
它的项Item被称为节点，用于树一样的层级关系

  • 表头：可以设置为QTreeWidgetItem对象，不仅可以设置表头的文字还可以设置字体、对齐方式、背景色、图标等。
    • 简单设置：可以使用setHeaderLabels()将字符串列表内容作为表头的标题
      void QTreeWidget::setHeaderLabels(const QStringList &labels)
    • 创建了QTreeWidgetItem作为表头，可以使用函数setHeaderItem()设置表头，还可以用headerItem()返回表头的对象指针。
      void QTreeWidget::setHeaderItem(QTreeWidgetItem *item) //设置表头节点
      QTreeWidgetItem *QTreeWidget::headerItem() //返回表头节点
  • 顶层节点：目录树里面一行就是一个节点，节点是QTreeWidgetItem对象，子节点就是下一个节点。以下是关于顶层节点的方法。
    • 添加(add)、插入(insert)、删除(take)、统计(count)、排序(sort)、返回位置(索引indexof)
    • int topLevelItemCount() //返回顶层节点的个数
    • void addTopLevelItem(QTreeWidgetItem *item) //添加一个顶层节点
    • void insertTopLevelItem(int index,QTreeWidgetItem *item) //插入一个顶层节点
    • int indexofTopLevelItem(QTreeWIdgetItem *item) //返回一个顶层节点的索引
    • QTreeWidgetItem *topLevelItem(int index) //根据index获取顶层节点
    • QTreeWidgetItem *takeTopLevelItem(int index) //根据索引移除一个顶层节点，但是并不是删除这个节点对象
  • QTreeWidget的其他接口函数
    • int columnCount() //返回表头列数
    • void setColumnCount(int columns) //设置表头列数
    • void sortItems(int column,Qt::SortOrder order) //将目录是按照某一列的表头排序
    • int sortColumn() //返回排序的列的序号
    • QTreeWidget *currentItem() //返回当前节点
    • QList<QTreeWidgetItem *>selectedItems() //返回选择的节点的列表
  • QTreeWidget公有槽函数
    • void clear() //清空整个目录树，表头除外
    • void collapaseItem(const QTreeWidgetItem *item) //折叠节点（对于所有节点）
    • void expandItem(const QTreeWidgetItem * item) //展开节点

QFileDialog的用法

在Qt中，QFileDialog 用于提供文件选择对话框，让用户可以选择文件或目录。
QFileDialog::getOpenFileName 是一个用于打开文件对话框并返回用户选择的文件路径的静态方法。以下是如何使用这个方法的详细说明和示例代码。

方法签名：

QString QFileDialog::getOpenFileName(QWidget *parent = nullptr, const QString &caption = QString(), const QString &directory = QString(), const QString &filter = QString(), QString *selectedFilter = nullptr, QFileDialog::Options options = QFileDialog::Options())

参数说明

  • parent: 父窗口指针，用于设置对话框的父级。
  • caption: 对话框的标题。
  • directory: 对话框打开时显示的初始目录。
  • filter: 过滤器，用于限制用户可以选择的文件类型，例如 “图片文件 (*.png *.jpg .bmp);;所有文件 ()”。
  • selectedFilter: 用于返回用户选择的过滤器（可选）。
  • options: 对话框选项，以支持不同的功能（例如，是否可以选择多个文件）。

QFileInfo

  • 检查文件或目录是否存在:
    exists() 方法检查文件或目录是否存在。

  • 获取文件名和路径:
    fileName() 获取文件名（不包括路径）。
    filePath() 获取文件的完整路径。
    absoluteFilePath() 获取文件的绝对路径。

  • 获取文件大小和类型:
    size() 返回文件的大小（以字节为单位）。
    suffix() 返回文件的扩展名。

  • 获取文件创建时间、修改时间:

QAbstractItemModel类和QAbstractItemView类

QAbstractItemModel类是所有模型类的直接或者间接的父类，定义了模型类的通用接口函数，对数据进行设置。
它是一个抽象类不能够初始化
QAbstractItemView类是所有视图组件的父类，它定义了一系列接口和函数

QAbstractItemModel接口函数

  • 行数和列数：rowCount()，columnCount()返回行和列
    • 对于树状模型，它需要设置parent父项模型索引
  • 插入和删除行：insertRow(),insertRows();removeRow(),removeRows()
  • 插入和删除列同上
  • 移动行和列：moveRow()、moveColumn()
  • 数据排序：sort()
  • 设置和读取项的数据：setData()、data()
    • setData()函数中需要QModelIndex index的参数传递才能设置数据
  • 清除一个项的数据：clearItemData()

正则表达式

正则表达式（Regular Expression）是一种强大的文本处理工具，它可以用来进行字符串的搜索、替换、分割和校验等操作。一个正则表达式模式是由普通字符（例如字母和数字）以及特殊字符（也称为元字符）组成的字符串，这些特殊字符具有特定的含义，可以用来指定复杂的匹配规则。

以下是一些常见的正则表达式模式及其说明：

基本匹配

abc：精确匹配字符串 “abc”。
a.c：匹配以 “a” 开头，以 “c” 结尾，中间是任意一个字符的字符串。
a*：匹配零个或多个 “a” 字符。
a+：匹配一个或多个 “a” 字符。
a?：匹配零个或一个 “a” 字符。

字符类

[abc]：匹配 “a”、“b” 或 “c” 中的任意一个字符。
[^abc]：匹配除了 “a”、“b”、“c” 之外的任意一个字符。
[a-z]：匹配从 “a” 到 “z” 的任意一个字符。
[0-9]：匹配任意一个数字字符。

预定义字符类

\d：匹配任意一个数字字符（等价于 [0-9]）。
\D：匹配任意一个非数字字符（等价于 [^0-9]）。
\w：匹配任意一个字母数字或下划线字符（等价于 [a-zA-Z0-9_]）。
\W：匹配任意一个非字母数字或下划线字符（等价于 [^a-zA-Z0-9_]）。
\s：匹配任意一个空白字符（空格、制表符、换行符等）。
\S：匹配任意一个非空白字符。

边界匹配器

分组和引用

(abc)：将 “abc” 分为一个捕获组。
(?:abc)：将 “abc” 分为一个非捕获组。
\n：引用第 n 个捕获组匹配的文本。

量词

{n}：恰好匹配 n 次。
{n,}：至少匹配 n 次。
{n,m}：至少匹配 n 次，但不超过 m 次。
选择符
a|b：匹配 “a” 或 “b”。

QPainter

QPainter 是 Qt 框架中用于绘制图形和文本的一个类。它可以在多种绘图设备上进行绘制，包括 QWidget、QPixmap、QImage、QPicture 等。以下是 QPainter 的一些基本用法：

创建 QPainter 对象

首先，你需要创建一个 QPainter 对象。通常，你会在绘图设备的 paintEvent() 方法中创建它。

绘制图形

QPainter 提供了多种方法来绘制基本图形。

  • 绘制线条
    painter.drawLine(QPoint(10, 10), QPoint(100, 100));
  • 绘制矩形
    painter.drawRect(10, 10, 100, 100); // 绘制一个矩形框
    painter.fillRect(10, 10, 100, 100, Qt::blue); // 填充矩形区域为蓝色
  • 绘制椭圆
    painter.drawEllipse(QPoint(50, 50), 50, 50); // 以点(50, 50)为中心，绘制半径为50的椭圆
  • 绘制多边形
    QPolygon polygon;
    polygon << QPoint(10, 10) << QPoint(100, 10) << QPoint(100, 100) << QPoint(10, 100);
    painter.drawPolygon(polygon);
  • 绘制文本
    你可以使用 QPainter 来绘制文本。

painter.drawText(10, 10, “Hello, World!”);

设置画笔和画刷

QPainter 使用 QPen 和 QBrush 来设置绘制图形的线条和填充样式。

QPen pen(Qt::red, 3, Qt::DashDotLine); // 创建一个红色的画笔，线宽为3，线型为DashDotLine
QBrush brush(Qt::blue); // 创建一个蓝色的画刷

painter.setPen(pen); // 设置画笔
painter.setBrush(brush); // 设置画刷
绘制图像
QPainter 可以用来绘制 QPixmap 或 QImage。

QPixmap pixmap(“path/to/image.png”);
painter.drawPixmap(10, 10, pixmap);

状态保存与恢复

在进行复杂的绘制操作时，你可以保存和恢复 QPainter 的状态。

painter.save(); // 保存当前状态
// 进行一些绘制操作
painter.restore(); // 恢复到保存的状态

事件系统

按照事件的来源，可以将事件分为三类：

  • 自生事件（spontaneous event）:由窗口系统产生，如：QKeyEvent、QMouseEvent
    自生事件会进入系统队列，等待事件循环的处理
  • 发布事件（posted event）:是由Qt应用程序产生，如：QTimerEvent，使用
    QCoreApplication::postEvent()产生发布事件，等待事件循环的处理。
  • 发送事件（sent event）:有Qt或其他程序定向发送的事件。使用
    QCoreApplication::postEvent()产生发送事件，有对象的event()函数直接处理

Qt的主事件循环(QCoreApplication::exec())从事件队列中获取原生的系统事件，将他们转换
为QEvent，并将转换后的事件发送给QObject。任何QObject派生的类都可以处理事件**

其中QPointEvent应该被成为点击事件

QWidget类重新实现了函数event()。并针对一些典型类型的事件定义了专门的事件处理函数。要对一些典型时间进行处理，只需要重新实现这些事件处理函数即可
