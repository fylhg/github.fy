QMainWindow和QAction

QMainWindow是主窗口，在对主窗口进行设计的时候，需要主菜单上的工具栏，菜单栏，按钮的下来菜单，组件的快捷菜单都需要用到QAction类,我们可以设计多个QAction创建菜单和工具按钮。

工具栏QToolBar类

通过QAction创建工具栏中的按钮，工具栏对应的就是QToolBar类。
在ui界面项快速创建工具栏可以右击鼠标然后点击”Add Tool Bar”即可。
QToolBar的主要属性：

属性名称 属性值类型 含义和作用
movable bool 工具栏是否可以动
allowedAreas Qt::ToolBarAreas 工具栏可以放置的窗口区域
orientation Qt::Orientation 工具栏的方向：水平、垂直
iconSize QSize 图标大小
toolButtonStyle Qt::ToolButtonStyle 工具按钮样式
floatavle bool 工具栏是否可浮动

ToolButtonStyle:

  • Qt::ToolButtonIconOnly:只显示图标
  • Qt::ToolButtonTextOnly:只显示文字
  • Qt::ToolButtonTextBesideIcon:文字显示在图标旁边
  • Qt::ToolButtonTextUnderIcon:文字显示在图标下面
  • Qt::ToolButtonFollowStyle:由QStyle样式定义

由于有些组件不能直接添加在工具栏中，只能通过QToolBar提供的函数接口来创建

  • void addAction(QAction*action)//添加一个Action，并根据Action设置自动创建按钮
  • QAction addWidget(QWidgetwidget)//添加一个界面组件
  • QAction insertWidget(QActionbefore,QWidget*widget)//插入一个界面组件
  • QAction *addSeperator()//添加一个分隔条
  • QAction *insertSeperator(QAction *before)//插入一个分隔条

QAction的信号

  • void hovered() //鼠标移动到此Action创建的菜单项或工具按钮上时
  • void toggled(bool checked) //checked属性变化时
  • void triggered(bool checked=false) //点击用此Action创建的菜单项或工具栏按钮时

QAction公有槽

  • void hover() //触发hovered()信号
  • void toggle() //反转checked属性
  • void trigger()//触发triggered()信号

QActionGroup

QActionGroup是 Qt 框架中的一个类，用于将多个 QAction 对象分组，使它们在用户界面中表现为互斥的选项。通常用于实现单选按钮组的功能，即用户只能选择其中的一个选项。
使用 QActionGroup 的步骤

  • 创建 QActionGroup 实例：
    你需要创建一个 QActionGroup 对象，用于管理一组 QAction 对象。

  • 创建 QAction 对象：
    创建多个 QAction 对象，并将它们QActionGroup->addAction()添加到 QActionGroup 中。

  • 将 QAction 添加到菜单或工具栏：
    将 QAction 对象添加到菜单或工具栏中，以便在用户界面中显示。

QFontComBox

  • 创建 QFontComboBox 实例：
    你需要创建一个 QFontComboBox 对象，并将其添加到你的界面中。

  • 连接信号和槽：
    可以将 QFontComboBox 的信号连接到自定义的槽，以响应用户选择字体的变化。

  • 设置和获取字体：
    可以通过 QFontComboBox 获取用户选择的字体，并进行相应的处理，比如应用到文本中。

  • 设置互斥行为：setExclusive()
    QActionGroup 自动将组内的 QAction 设为互斥的，即同一时间只能选择一个 QAction。

总结
Item Views：基于模型/视图架构，适用于需要高效处理大量数据的场景。视图和数据分离，适合复杂的数据展示和交互需求。
Item Widgets：直接在视图中嵌入小部件，适用于数据量不大且需要复杂小部件的场景。视图和数据紧密结合，操作简单直观。

QToolBox组件

QToolBox是一个工具箱组件类，工具箱是一种支持垂直分页的多页容器组件
使用Insert Page和Delete Page添加菜单和删除。
工具箱的每一个页面就是一个QWidget组件

QToolBox有一个信号：

  • void QToolBox::currentChanged(int index)//在切换当前页面组件的时候发射这个信号

QToolButton属性

  • popupMode属性
    • 属性值是：枚举类型QToolButton::ToolButtonPopupMode,当按钮有下拉菜单的时候，这个属性决定了菜单的模式
    • QToolButton::DelayedPopup:按下按钮过一会才会显示下拉菜单
    • QToolButton::MenuButtonPopup:会在按钮右侧给一个箭头，按下显示菜单，执行绑定的Action
    • QToolButton::InstanPopup:会在按钮右侧给一个箭头，按下立刻显示菜单，不会执行绑定的Action
  • toolButtonStyle属性
    • 属性值是：Qt::QToolButtonStyle,表示工具按钮上的图标和文字显示方式
  • autoRaise属性
    • 属性值是：设置边框，设置为true那么就没有边框
  • arrowType属性：
    • 属性值是：Qt::ArrowType有没有箭头

QToolButton其他接口函数

  • setDefaultAction()函数：
    • 用于为工具按钮绑定Action，绑定以后工具栏按钮的文本，图标等与Action一样，
    • 工具按钮的trigger()信号自动关联Action的trigger()函数
    • 代码：void QToolBox::setDefaultAction(QAction*action)
  • setMenu()函数
    • 用于为工具按钮设计下拉菜单
    • 代码：void QToolBox::setMenu(QMenu*menu)
  • setIcon()函数
    • 用来给toolButton设置图标
  • setText()函数
  • setToolButtonStyle()函数
    • 设置toolButton的样式
    • QToolButton::DelayedPopup:按下按钮过一会才会显示下拉菜单
    • QToolButton::MenuButtonPopup:会在按钮右侧给一个箭头，按下显示菜单，执行绑定的Action
    • QToolButton::InstanPopup:会在按钮右侧给一个箭头，按下立刻显示菜单，不会执行绑定的Action
  • setPopUpMode()函数

QListWidget

QListWidgetItem

QListWidgetItem是在设置QListWidget的每一项，每一项都是QListWidgetItem对象。

然后QListWidgetItem提供了一系列函数对每一项的Text,Icon,Data,Flags,CheckState,Selected进行设置

  • setText()
  • setIcon(QIcon&icon) 需要先用QIcon icon定义图标配置资源文件路径icon->addFile(“文件路径”)
  • setData() 注意下获取到的数据是QVariant类型的
  • setFlags() 设置项的特性Qt::ItemFlags的枚举类型
    • Qt::ItemIsEditable 允许编辑
    • Qt::ItemIsSelectable 允许选中
    • Qt::ItemIsUserCheckable 前面会加一个选中的复选框
    • Qt::ItemIsEnabled 是否可用

QListWidget提供的借口函数

添加或删除项：

  • addItem()和addItems()
  • insertItem()和insertItems()
  • QListWidgetItem*takeItem() 从列表组件中删除一个项，并返回这个项的对象指针，要想完全删除得把这个指针也删了
  • clear() 全删了
    项的访问：
  • QListWidgetItem *currentItem()
  • void setCurentItem() 设置当前项
  • QListWidgetItem*item() 根据行号返回一个item
  • int currentRow() 返回当前的行号，-1表示没有
  • void setCurentRow 设置当前行
  • int row() 返回一个项所在的行号
  • int count() 返回列表中项的个数
    排序：
  • void setSortingEnable() 设置列表是否可排序
  • void isSortingEnabled() 列表是否可排序
  • void sortItems() 对列表进行排序，可指定为升序或者降序

创建右击快捷菜单

每个继承QWidget的类都有customContextMenuRequested()信号
要使QWidget的组件可以发射这个信号需要设置其contextMenuPolicy属性：

  • void QWidget::setContextMenuPolicy
    有以下几种枚举值：
    • Qt::NoContextMenu:组件没有快捷菜单，由父容器处理
    • Qt::PreventContextMenu:阻止快捷菜单，并且右击鼠标事件不会交给父容器处理
    • Qt::DefaultContextMenu:默认快捷菜单
    • Qt::ActionsContextMenu:自动根据QWidget::actions()返回的Action创建并显示快捷菜单
    • Qt::CustomContextMenu:组件发射 customContextMenuRequested()信号，自己写槽函数创建菜单

QTreeWidget

QTreeWidget是一种Item widget组件，类似的可以参考QListWidget它也是Item widget组件。
它的项Item被称为节点，用于树一样的层级关系

  • 表头：可以设置为QTreeWidgetItem对象，不仅可以设置表头的文字还可以设置字体、对齐方式、背景色、图标等。
    • 简单设置：可以使用setHeaderLabels()将字符串列表内容作为表头的标题
      void QTreeWidget::setHeaderLabels(const QStringList &labels)
    • 创建了QTreeWidgetItem作为表头，可以使用函数setHeaderItem()设置表头，还可以用headerItem()返回表头的对象指针。
      void QTreeWidget::setHeaderItem(QTreeWidgetItem *item) //设置表头节点
      QTreeWidgetItem *QTreeWidget::headerItem() //返回表头节点
  • 顶层节点：目录树里面一行就是一个节点，节点是QTreeWidgetItem对象，子节点就是下一个节点。以下是关于顶层节点的方法。
    • 添加(add)、插入(insert)、删除(take)、统计(count)、排序(sort)、返回位置(索引indexof)
    • int topLevelItemCount() //返回顶层节点的个数
    • void addTopLevelItem(QTreeWidgetItem *item) //添加一个顶层节点
    • void insertTopLevelItem(int index,QTreeWidgetItem *item) //插入一个顶层节点
    • int indexofTopLevelItem(QTreeWIdgetItem *item) //返回一个顶层节点的索引
    • QTreeWidgetItem *topLevelItem(int index) //根据index获取顶层节点
    • QTreeWidgetItem *takeTopLevelItem(int index) //根据索引移除一个顶层节点，但是并不是删除这个节点对象
  • QTreeWidget的其他接口函数
    • int columnCount() //返回表头列数
    • void setColumnCount(int columns) //设置表头列数
    • void sortItems(int column,Qt::SortOrder order) //将目录是按照某一列的表头排序
    • int sortColumn() //返回排序的列的序号
    • QTreeWidget *currentItem() //返回当前节点
    • QList<QTreeWidgetItem *>selectedItems() //返回选择的节点的列表
  • QTreeWidget公有槽函数
    • void clear() //清空整个目录树，表头除外
    • void collapaseItem(const QTreeWidgetItem *item) //折叠节点（对于所有节点）
    • void expandItem(const QTreeWidgetItem * item) //展开节点

QFileDialog的用法

在Qt中，QFileDialog 用于提供文件选择对话框，让用户可以选择文件或目录。
QFileDialog::getOpenFileName 是一个用于打开文件对话框并返回用户选择的文件路径的静态方法。以下是如何使用这个方法的详细说明和示例代码。

方法签名：

QString QFileDialog::getOpenFileName(QWidget *parent = nullptr, const QString &caption = QString(), const QString &directory = QString(), const QString &filter = QString(), QString *selectedFilter = nullptr, QFileDialog::Options options = QFileDialog::Options())

参数说明

  • parent: 父窗口指针，用于设置对话框的父级。
  • caption: 对话框的标题。
  • directory: 对话框打开时显示的初始目录。
  • filter: 过滤器，用于限制用户可以选择的文件类型，例如 “图片文件 (*.png *.jpg .bmp);;所有文件 ()”。
  • selectedFilter: 用于返回用户选择的过滤器（可选）。
  • options: 对话框选项，以支持不同的功能（例如，是否可以选择多个文件）。

QFileInfo

QFileInfo 是 Qt 中的一个类，用于获取文件或目录的详细信息。它可以帮助你检查文件或目录的存在性、获取路径、文件大小、创建时间等信息。以下是 QFileInfo 的使用方法和示例代码。

  • 检查文件或目录是否存在:
    exists() 方法检查文件或目录是否存在。

  • 获取文件名和路径:
    fileName() 获取文件名（不包括路径）。
    filePath() 获取文件的完整路径。
    absoluteFilePath() 获取文件的绝对路径。

  • 获取文件大小和类型:
    size() 返回文件的大小（以字节为单位）。
    suffix() 返回文件的扩展名。

  • 获取文件创建时间、修改时间:
    created() 返回文件创建时间。
    lastModified() 返回文件最后修改时间。

  • 检查是否是目录或文件:
    isDir() 检查是否是目录。
    isFile() 检查是否是文件。
